# Hebrew RTL Artifact

מטרת הסקיל: לוודא שכל Artifact (HTML) עם תוכן בעברית נבנה עם RTL תקין מההתחלה, ולא כתוספת בדיעבד.

## חוקי יסוד

1. על אלמנט השורש של התוכן (למשל ה-`<div>` העוטף) קבעו `dir="rtl"` ו-`lang="he"`.
2. פונט קריא לעברית: העדיפו font stack עם תמיכה עברית טובה, לדוגמה:
   ```css
   font-family: "Segoe UI", Arial, "Noto Sans Hebrew", sans-serif;
   ```
3. יישור טקסט: `text-align: right` כברירת מחדל בגוף הטקסט (לא `left`).
4. Flexbox / Grid: כיוון `rtl` הופך אוטומטית את סדר הפריטים ב-`flex-direction: row`, אך יש לבדוק חזותית שהסדר ההגיוני (מימין לשמאל) נכון - במיוחד בתפריטים, breadcrumbs וכפתורי ניווט (הבא/הקודם).
5. מספרים, קוד, כתובות אימייל ו-URL תמיד נשארים LTR בתוך טקסט RTL - זה קורה אוטומטית ב-HTML/CSS רגיל, אין צורך להתערב, אך אם יש בעיה אפשר לעטוף ב-`<span dir="ltr">`.
6. אייקונים כיווניים (חיצים, "הבא/הקודם", גרירה): יש להפוך אותם כשעוברים ל-RTL (חץ שהיה שמאלה הופך ימינה וכו').
7. תמיכה גם ב-light וגם ב-dark מצב (לפי כללי ה-artifact הרגילים), בנוסף ל-RTL.

## תבנית התחלתית

```html
<div dir="rtl" lang="he" style="font-family: 'Segoe UI', Arial, 'Noto Sans Hebrew', sans-serif; text-align: right;">
  <!-- תוכן העמוד כאן -->
</div>
```

## בדיקה לפני מסירה

- לגלול על העמוד ולוודא שהטקסט מיושר לימין ולא "קופץ" שמאלה.
- לוודא שרשימות, טבלאות וכרטיסיות זורמות מימין לשמאל ולא משאירות סדר LTR "דבוק".
- אם יש גרפים/צ'ארטים - הצירים והמקרא יכולים להישאר LTR (זו קונבנציה מקובלת), אבל הכותרות וההסברים סביבם - בעברית ו-RTL.
